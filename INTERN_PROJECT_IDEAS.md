### 1. 🧪 FX Trade Test Coverage Visualizer
**Type:** QA / Data Visualization  
**Tools:** Python, pandas, matplotlib/plotly (all free)  
**Time:** 1–2 weeks

**What it is:**  
A dashboard that visualizes test coverage gaps across FX trade types (spot, forward, outright). You design mock trade scenarios and map which test cases cover which flows.

**Why it's non-invasive:**  
Uses no real Visa data — you build it with synthetic/mock trade data you create yourself.

**What you learn:**  
Test matrix design, coverage gap analysis, how to communicate QA health to PMs and engineers visually.

**Deliverable:**  
A GitHub repo + demo that shows: "here's how I'd map test coverage across FX settlement types." Perfect portfolio piece.

### 4. 🔁 FX Settlement Simulation (Educational)
**Type:** Systems Design / Education  
**Tools:** Python (pure stdlib — no external deps)  
**Time:** 2–3 weeks

**What it is:**  
A Python simulation of an FX settlement pipeline:
- Generate random trade orders (spot, forward, outright)
- Validate trade parameters (counterparty, notional, settlement date)
- Route to correct settlement path (T+0, T+2, future date)
- Log trade events with status (PENDING → CONFIRMED → SETTLED → FAILED)

**Why it's non-invasive:**  
Completely fictional/educational. No Visa IP involved. Shows you understand the domain.

**What you learn:**  
State machine design. Event-driven architecture. How to write test cases for financial workflows.

**Deliverable:**  
A GitHub repo with full test suite (`pytest`). README explains the FX settlement lifecycle. Demonstrates deep domain knowledge without revealing anything proprietary.
